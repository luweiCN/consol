# I18N

ConSol 当前支持 `en-US` 和 `zh-CN`。所有用户可见文案必须来自 catalog，TUI 文案尤其不能硬编码在组件里。

## Locale Resolution

语言优先级：

1. 显式配置语言，且不是 `system`。
2. `CONSOL_LANG`
3. `LANGUAGE`
4. `LC_ALL`
5. `LC_MESSAGES`
6. `LANG`
7. 默认 `en-US`

`system` 表示继续走环境变量 fallback。

## Catalog Rules

- 新 key 必须同时加入 `en-US` 和 `zh-CN`。
- 两个语言的 placeholder 必须一致，例如 `{target}`、`{panel}`。
- 删除 UI surface 时同步删除不用的 key。
- 机器稳定字段不要随语言变化，用户可见文本才本地化。

## TUI Rules

- TUI snapshot 必须同时跑英文和中文。
- CJK 宽度必须通过 resize 或字符帧测试覆盖。
- 新增 selector、input、modal、scrollbar 时，同时覆盖键盘和鼠标路径。

## Commands

```bash
bun run check:i18n
bun run check:no-inline-ui-copy
bun run test:tui
```
